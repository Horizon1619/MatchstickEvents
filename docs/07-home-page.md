# Purpose

The homepage is the most important page of the website.

Its objective is **not** to explain everything.

Instead, it should:

- Create an unforgettable first impression.
- Build trust immediately.
- Showcase the company's craftsmanship.
- Inspire visitors emotionally.
- Encourage consultation requests.

The homepage should function as the digital front entrance to the MatchStick Events brand.

---

# Homepage Objectives

Business Goals

- Establish credibility.
- Showcase premium quality.
- Increase consultation requests.
- Encourage exploration.
- Build emotional connection.

User Goals

Visitors should quickly understand:

- Who MatchStick Events is.
- What services are offered.
- Whether they trust the company.
- Whether they want to enquire.

The website should communicate luxury, creativity, trust, sophistication, attention to detail, and personalized event experiences. 0

---

# Information Hierarchy

The homepage should communicate information in the following order:

1. Emotional impact
2. Brand identity
3. Value proposition
4. Portfolio quality
5. Services
6. Trust
7. Consultation

Never reverse this order.

# Mobile Behaviour

The homepage must remain fully usable on devices from 320px width onwards.

Requirements:

- Single-column layouts where appropriate.
- Large touch targets.
- Responsive typography.
- Optimized media.
- No horizontal scrolling.

---

# Tablet Behaviour

Tablet layouts should preserve the desktop visual hierarchy while reducing the number of columns.

Preferred layout:

Two-column sections wherever practical.

---

# Desktop Behaviour

Desktop should provide:

- Maximum visual impact.
- Generous whitespace.
- Cinematic imagery.
- Smooth scrolling.
- Large typography.
- Immersive presentation.

---

# Scroll Behaviour

The homepage should encourage continuous exploration.

Preferred effects:

- Fade-in sections.
- Subtle parallax.
- Image reveal.
- Text reveal.
- Sticky navigation transition.

Avoid:

- Excessive motion.
- Long animation delays.
- Distracting effects.

---

# Loading States

While content loads:

Display:

- Skeleton cards.
- Skeleton text.
- Placeholder hero image.
- Smooth fade transition.

Avoid empty white screens.

---

# Error States

If homepage content fails to load:

Display:

- Friendly message.
- Retry button.
- Contact link.

The website should never expose technical errors to visitors.

---

# Empty States

If dynamic sections (such as featured events or Instagram feed) have no content:

Display elegant placeholder messaging instead of leaving blank areas.

Examples:

- "New Celebrations Coming Soon"
- "More Moments Will Be Shared Here"

# Accessibility Checklist

The homepage must satisfy:

- Semantic HTML.
- Proper heading hierarchy.
- Keyboard navigation.
- Screen reader compatibility.
- Alt text for every meaningful image.
- Visible focus indicators.
- WCAG AA colour contrast.
- Minimum touch target size of 44×44px.

---

# Performance Goals

Target:

- First Contentful Paint under 2.5 seconds.
- Largest Contentful Paint under 2.5 seconds.
- Cumulative Layout Shift below 0.1.
- Lighthouse Performance Score above 90.
- Lighthouse Accessibility Score above 95.

These are implementation targets recommended for a premium website and are not requirements from the client brief.

# Functional Requirements

The homepage shall:

- Display company branding.
- Present navigation.
- Showcase selected portfolio items.
- Introduce primary services.
- Link to major pages.
- Encourage consultation requests.
- Display contact access.
- Maintain responsive behaviour.

---

# Non-Functional Requirements

The homepage shall be:

- Responsive.
- Accessible.
- Fast.
- Secure.
- Maintainable.
- SEO friendly.
- Consistent with the design system.

# Developer Notes

Developers should:

- Build reusable homepage sections.
- Follow the design tokens defined in `06-design-system.md`.
- Avoid hardcoded spacing and colours.
- Optimize media assets.
- Lazy-load non-critical images.
- Ensure smooth but restrained animations.

---

# Acceptance Criteria

The homepage implementation is complete when:

- All required homepage sections are implemented.
- Navigation functions correctly.
- Primary CTA is visible and functional.
- Featured portfolio links correctly.
- Services preview links correctly.
- Footer contains all required information.
- Responsive layouts work across supported breakpoints.
- Accessibility checks pass.
- Performance targets are reasonably achieved.
- Visual design matches the established brand guidelines.
