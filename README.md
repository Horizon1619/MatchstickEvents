# MatchStick Events Website

> Documentation Repository v1.0

---

# Project Overview

This repository contains the complete functional, technical, design, and architectural documentation for the official website of **MatchStick Events**, a luxury event planning company based in Kolkata, India. The company was founded in 2012 by **Preksha Chhajer** and specializes in creating premium, elegant, and highly personalized events. The source document describes the company, branding preferences, services, portfolio, contact information, and desired website pages. 0 1

This repository serves as the **single source of truth** for designing, developing, testing, and maintaining the website.

It is intended for:

- Developers

- AI Coding Assistants (Gemini, ChatGPT, Claude, etc.)

- UI/UX Designers

- Future Contributors

- Project Owner

---

# Project Goal

Build a modern, elegant, premium website that reflects the identity of MatchStick Events.

The website should communicate:

- Luxury

- Creativity

- Trust

- Sophistication

- Attention to Detail

- Personalized Event Experiences

The website should feel more like a premium lifestyle brand than a traditional event management website.

---

# Primary Business Objectives

The website exists to:

- Build credibility.

- Showcase previous work.

- Increase consultation requests.

- Generate qualified leads.

- Strengthen the luxury brand identity.

- Create an exceptional first impression.

The website is **not** intended to sell predefined event packages or display public pricing.

---

# Target Audience

The website is primarily designed for:

- Couples planning weddings

- Families organizing milestone celebrations

- Parents planning birthdays or baby showers

- Corporate organizations

- Luxury private clients

- High-income professionals

- Individuals looking for bespoke event experiences

---

# Core Philosophy

Every page should make visitors feel:

> "If they can create experiences this beautiful, I want them to create mine."

The experience should emphasize emotion before information.

---

# Design Principles

The website should always prioritize:

- Elegance

- Minimalism

- Premium aesthetics

- High-quality photography

- Spacious layouts

- Smooth animations

- Excellent readability

- Fast loading

- Mobile-first responsiveness

- Accessibility

Avoid:

- Visual clutter

- Cheap-looking graphics

- Excessive colors

- Flashy effects

- Generic templates

These principles are consistent with the client's preference for a modern, elegant look that is not messy or filled with out-of-context colours. 2
